Flujos de interacción detallados
Flujo de autenticación
1
Médico rellena formulario → POST /register con Pydantic RegisterRequest
2
Backend genera token urlsafe(32) → INSERT en solicitudes_registro
3
Email al admin vía Brevo SMTP (puerto 2525) con enlace /approve/{token}
4
Admin clica → GET /approve → crea usuario con pw = token_urlsafe(10) hasheada con bcrypt
5
Email al médico con credenciales → POST /login → verify_password bcrypt → sessionStorage
Flujo de evaluación clínica
1
AntecedentesView: NHC, fecha nacimiento, género → guardado en estado App.jsx
2
EventRecordView: desencadenante, lugar, actividad, tratamiento → estado App.jsx
3
CalculadoraView: selección síntomas por sistema orgánico → POST /calculate con x-tfg-key
4
Backend: SHA-256(NHC) → pseudónimo · get_age_range() · cifrado Fernet de campos clínicos
5
logic.py: nFASS = log₂(Σλ·ε)+2 → oFASS grade 0-4 · INSERT/UPDATE registros → devuelve id_registro
Flujo ChatBot IA
1
Médico escribe pregunta en ChatBot.jsx → GET /chat?user_message=...
2
Backend carga SYSTEM_PROMPT de agent_logic.py (restricción dominio alergología/FASS)
3
Groq SDK → modelo llama-3.3-70b-versatile con timeout 25 s
4
Respuesta texto → frontend renderiza en burbuja de chat
★
Endpoint GET /chat no requiere x-tfg-key (datos no clínicos)
🔒 Seguridad CORS: 3 orígenes x-tfg-key → HTTP 401 bcrypt passwords Fernet AES-128-CBC SHA-256 NHC 403 autoría Pydantic validation sessionStorage token urlsafe(32)
